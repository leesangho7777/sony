## **👨‍🏫 프로젝트 소개**

소니 웹사이트를 **모바일 퍼스트(Mobile-First)** 전략으로 리뉴얼한 반응형 웹페이지 프로젝트입니다. 

**모바일 퍼스트(Mobile-First)** 전략을 기반으로 **기준 너비 360px**에서 시작하여 다양한 디바이스 환경에 대응하도록 설계하였습니다.

**사용자의 주요 목적**이 ‘상품 탐색 및 구매’임을 고려하여, 홈 화면을 중심으로 **직관적이고 스토리 있는** 탐색 경험을 제공합니다.

## 🎨 디자인 및 사용자 흐름 설계 (UX/UI Planning)

홈 화면은 사용자가 웹사이트를 방문하는 **핵심 목적이 ‘상품 탐색 및 구매’**라는 점에 초점을 맞추어 구성했습니다.

따라서 첫 진입 시 시각적으로 주목할 수 있는 **상품 중심 메인 슬라이드 배너**를 배치하였으며, 그 하단에는

- **검색창(Search Bar)** 과
- **카테고리 기반 상품 탐색 영역(Article Layout)**
    
    을 구성하여 사용자가 원하는 상품을 **최대한 직관적으로 탐색할 수 있도록 설계**했습니다.
    

이후 메인 페이지는 단순한 제품 나열이 아닌, **스토리가 있는 내러티브 구조(Storytelling Layout)** 로 디자인하였습니다.

사용자가 자연스럽게 페이지를 스크롤하며 **브랜드의 감성적 메시지와 제품의 핵심 가치**를 함께 경험할 수 있도록 하여, **시각적 몰입감과 사용 편의성을 동시에 확보**했습니다.

---

## 🧩 서브 페이지 구성 (Sub Pages)

- **로그인(Login)** 및 **회원가입(Sign-up)** 페이지는
    
    UI 시스템의 일관성을 유지하기 위해 **통일된 컴포넌트 디자인과 CSS 모듈 구조**를 적용했습니다.
    
- **문의하기(Contact)** 페이지 또한 동일한 인터페이스 규칙(UI Guideline)에 맞추어 구성함으로써,
    
    사용자가 어떤 페이지에 머무르더라도 **일관된 사용 경험(Consistent UX)** 을 제공하도록 설계했습니다.
    

이를 통해 프로젝트 전반에서 **통합된 UI 시스템**을 구현하였으며, 각 페이지가 독립적이면서도 조화롭게 연결되도록 구성했습니다.

## ⚙️ 반응형 구현 (Responsive Web Implementation)

모든 레이아웃은 **CSS 미디어쿼리(Media Query)** 를 활용하여 다양한 해상도(모바일, 태블릿, PC)에 대응하도록 구성했습니다.

디자인 요소의 크기, 여백, 폰트, 이미지 비율 등을 **상대 단위(rem, %, vw 등)** 로 정의해, 화면 크기에 따라 유연하게 조정되도록 하였습니다.

또한, **JavaScript를 활용한 인터랙티브 요소**를 적극적으로 구현하였습니다.

아래의 메서드들을 통해 페이지가 반응형 웹사이트로서의 동적 기능을 충분히 수행하도록 설계했습니다.

- `querySelector`/`getElementById` 등으로 DOM 선택 및 `classList.toggle`을 통한 UI 상태 제어
- `addEventListener` 기반 이벤트 바인딩으로 메뉴, 팝업, 아코디언, 슬라이드 등 인터랙션 구현
- `style` 조작과 `scrollBy` + `setTimeout` 활용한 동적 레이아웃 및 스크롤 제어

이를 통해 화면의 크기 변화뿐만 아니라, 사용자의 **스크롤, 클릭, 토글 등 다양한 인터랙션에 즉각 반응**할 수 있는 구조를 완성했습니다.

## 🧠 기술 이해 및 시스템 완성도

프로젝트 전반에 걸쳐 **UI 시스템 설계, CSS 구조화, JavaScript 이벤트 핸들링**에 대한 깊은 이해를 기반으로 작업을 진행했습니다.

디자인 일관성을 유지하면서도, 유지보수가 용이하도록 **클래스 기반의 구조화된 코드 스타일**을 적용했습니다.

또한, 모든 구성 요소가 반응형으로 작동하도록 설계함으로써

“어떤 디바이스에서도 동일한 브랜드 경험을 제공한다”는 웹 접근성의 기본 원칙을 충실히 구현했습니다.
